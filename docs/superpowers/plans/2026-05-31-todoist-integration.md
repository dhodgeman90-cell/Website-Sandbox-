# Todoist Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the inbox.md task workflow with Todoist, connected to Claude Code via the official Todoist MCP server, so tasks added on any device are automatically read at every session start.

**Architecture:** The official Todoist HTTP MCP server (`https://ai.todoist.net/mcp`) is registered with Claude Code using a single CLI command. Claude reads open tasks from the dedicated "VS Code" Todoist project at session start, grouped by section, and marks them complete via MCP when work is done. No middleware, no syncing, no extra tooling.

**Tech Stack:** Claude Code CLI (`claude mcp add`), Todoist HTTP MCP server, Todoist app (phone + desktop)

---

### Task 1: Add the Todoist MCP Server to Claude Code

**Files:**
- Modifies: `~/.claude/settings.json` (handled automatically by the claude CLI — you do not edit this file manually)

This step connects Claude Code to Todoist's official MCP server. You only need to do this once — it persists across all future sessions.

- [ ] **Step 1: Run the add command**

In PowerShell, run:
```powershell
claude mcp add --transport http todoist https://ai.todoist.net/mcp
```

Expected output: a confirmation message saying the server was added, plus a note like `"1 MCP server needs auth · /mcp"`

- [ ] **Step 2: Authenticate with Todoist**

Launch Claude Code in this project folder:
```powershell
claude
```

Then inside Claude Code, type:
```
/mcp
```

Use the arrow keys to select **Todoist** from the list, press Enter, and complete the OAuth login that opens in your browser. Log in with your Todoist account credentials.

- [ ] **Step 3: Verify the connection**

Still inside Claude Code, type:
```
/mcp
```

Todoist should now show as **connected** (no auth warning). If it still shows as needing auth, close and reopen Claude Code and try `/mcp` again.

- [ ] **Step 4: Commit nothing — this task modifies only your local Claude config, not the repo**
